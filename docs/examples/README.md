# Usage Examples

Real-world scenarios demonstrating Token Audit capabilities. Each example includes a problem statement, step-by-step solution, and key takeaways.

---

## Examples

| Example | Scenario | Key Skills |
|---------|----------|------------|
| [Debugging a Slow Session](debugging-slow-session.md) | Find expensive tools causing auto-compaction | TUI analysis, tool ranking |
| [Optimizing MCP Config](optimizing-mcp-config.md) | Remove unused MCP servers | Context tax, zombie tools |
| [Comparing Platform Costs](comparing-platform-costs.md) | Claude vs Codex vs Gemini | Cross-platform reports |
| [Team-Wide Tracking](team-wide-tracking.md) | Multi-project aggregation | Configuration sharing |
| [CI/CD Integration](ci-cd-integration.md) | Automated cost monitoring | Headless mode, JSON output |
| [AI-Assisted Review](ai-assisted-review.md) | Using AI export effectively | AI export, analysis prompts |

---

## Prerequisites

All examples assume:
- Token Audit installed (`pipx install token-audit`)
- At least one supported platform (Claude Code, Codex CLI, or Gemini CLI)
- Familiarity with [Getting Started](../getting-started.md)
